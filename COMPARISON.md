## ⚙️ 配置方式对比

### 原版配置方式

**修改工作流文件：**
```yaml
# 需要编辑 .github/workflows/ztc-kernel.yml
CONFIG="android12-5.10-lts"  # 硬编码
LTO=thin                      # 硬编码
KCFLAGS="-O3"                # 硬编码
```

**缺点：**
- 需要修改代码
- 不支持 UI 配置
- 每次修改需要提交
- 不易于版本管理

### 增强版配置方式

**通过 GitHub Actions UI：**
```
1. 点击 Actions
2. 选择 ZTC Kernel Build (Enhanced)
3. 点击 Run workflow
4. 在 UI 中选择所有参数
5. 点击运行
```

**优点：**
- 无需修改代码
- 直观的 UI 界面
- 支持保存配置
- 易于版本管理

## 📝 常见问题

### Q: 增强版会替代原版吗？

**A:** 不会。两个版本会同时维护：
- 原版：保持简单稳定
- 增强版：提供更多功能

### Q: 可以同时使用两个版本吗？

**A:** 可以。它们是独立的工作流，互不影响。

### Q: 增强版的构建产物和原版兼容吗？

**A:** 完全兼容。使用相同配置时，产物是一致的。

### Q: 增强版会更不稳定吗？

**A:** 不会。增强版只是提供了更多选项，使用默认配置时和原版一样稳定。

### Q: 如何在两个版本之间切换？

**A:** 在 Actions 页面选择对应的工作流即可，无需其他操作。
